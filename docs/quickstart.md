# Quickstart

## Goal

This quickstart is the fastest path to a local run where:

- the server starts successfully
- the GUI launches
- command/response traffic works
- status updates appear in the GUI

## 1. Build the project

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Debug
cmake --build build
```

## 2. Choose a configuration file

For local setup, start from the repository example:

```text
Config/rimokun.yaml
```

If you are not ready to use hardware-backed settings, create a local variant and adjust environment-specific values before running against production equipment.

## 3. Start the server

Run the server and point it at the config file:

```bash
./build/Server/apps/rimoServer --config Config/rimokun.yaml
```

If your build layout differs, use the actual path of the `rimoServer` binary generated by CMake.

## 4. Launch the GUI

In a second terminal, start the GUI with the same configuration:

```bash
./build/GUI/rimokunControl --config Config/rimokun.yaml
```

Again, adjust the binary path if your local build output differs.

## 5. Confirm connectivity

At this stage you should confirm:

- the server stays running after initialization
- the GUI opens without reporting missing config
- status indicators begin updating instead of remaining stale
- the GUI does not report immediate connection loss

## 6. Confirm command and status behavior

Use a low-risk GUI action that sends a command and observe both sides:

- submit one operator action from the GUI
- check that the server logs a command receipt or related activity
- confirm the GUI receives a response
- confirm the next status update reflects the expected change or acknowledgement

## First successful usage example

An initial end-to-end validation can be:

1. Start `rimoServer`.
2. Start `rimokunControl`.
3. Open a panel that exposes an operator command, such as a reset or tool changer action.
4. Execute the action in a safe environment.
5. Confirm the GUI shows updated component or actuator state.

If you can complete that cycle without timeouts, stale status, or validation errors, the basic system path is working.
